# ADR 0009: v0.0.2 Platform Contracts and Compatibility Boundaries

Status: Accepted

## Decision

Hostwright uses the early `v0.0.2` window to introduce these breaking contracts together:

- Manifest v2;
- Control API v2;
- Runtime Provider API v2;
- plugin ABI v1;
- state schema v7.

The central source of truth is `HostwrightContractVersions`. Golden tests, CLI capability output, schema migrations, docs, and providers derive from or assert those values. Manifest v1/versionless input has a deterministic read-only migration preview. There is no silent API downgrade, provider takeover, ambient-privilege plugin shim, or state downgrade.

## Product Scope Decision

ADRs 0005 and 0006 are superseded. Kubernetes and broad Compose/Docker interoperability are owned by Phases 12 and 13. GUI, networking, cloud, multi-host, accelerator, plugin, and external compatibility work are similarly named roadmap outcomes, not permanent non-goals.

Permanent exclusions are limited to private Apple APIs, unsupported Intel/old-macOS emulation, unsafe writes without cluster quorum, silent telemetry, unauthenticated public control, and destructive garbage collection of unmanaged resources. Each user outcome receives a public-API or safe-mode fallback where possible.

## Compatibility Policy

- Current-main documentation describes only the development build and evidence-backed capability state.
- Immutable release notes keep their original claims and are visibly historical.
- At GA, Control API, Kubernetes, Docker, client, macOS, and Apple runtime support is an exact published matrix.
- API N/N-1 support begins only after v2 becomes a released baseline.
- One exact Containerization version is pinned per Hostwright release.
- Unsupported fields and endpoints return stable explicit errors; they are never silently ignored.

## Consequences

All later phases build on one identity, migration, capability, and evidence vocabulary. Breaking a locked contract requires an ADR update, migration path, golden-contract update, compatibility review, and evidence across every consumer.
