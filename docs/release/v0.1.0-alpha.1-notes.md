# Hostwright v0.1.0-alpha.1

Hostwright is a Mac-native desired-state control plane for Apple container workloads.

This is a source-only alpha pre-release. It is not production ready.

## What Works

- Swift Package Manager build for `hostwright` and `hostwrightd`.
- `hostwright --version`, `init`, `validate`, `plan`, `doctor`.
- Restricted `hostwright.yaml` manifest parsing and validation.
- Deterministic plan and drift output.
- Explicit SQLite state database paths.
- RuntimeAdapter-backed Apple container observation for verified output shapes.
- Narrow `hostwright apply` support for one create-missing-service action or one restart-policy-allowed managed start action.
- `hostwright status --state-db <path>`.
- Bounded `hostwright logs <service> --tail <n>` through RuntimeAdapter.
- `hostwright events --state-db <path>`.
- Ownership-scoped `hostwright cleanup` with dry-run and exact token confirmation.

## Compatibility

| Area | Status |
| --- | --- |
| CPU | Apple silicon only |
| OS | macOS 26 or newer |
| Build toolchain | Swift 6.2+ through Xcode/full developer tools |
| Runtime | Apple container CLI required for runtime observation and mutation |
| Package manager | Swift Package Manager |
| Release artifact | Source only |

## Install From Source

```bash
git clone https://github.com/hostwright/hostwright.git
cd hostwright
git checkout v0.1.0-alpha.1
swift build
swift test
swift run hostwright --version
```

No binary, installer package, Homebrew formula, signed artifact, or notarized artifact is provided for this alpha.

## Security And Safety Notes

- Runtime behavior goes through `RuntimeAdapter`.
- `apply` requires explicit `--state-db` and `--confirm-plan`.
- Cleanup requires explicit `--state-db`, dry-run, confirmation token, ownership records, live observation, exact Hostwright-owned container identifiers, and non-running lifecycle state.
- Cleanup does not delete images, volumes, networks, or unmanaged containers.
- Logs, events, runtime output, and state payloads use redaction, but redaction is heuristic and may not cover every secret format.

## Known Limitations

- Not production ready.
- No daemon reconciliation loop.
- `hostwrightd` is a scaffold only.
- No default state database path.
- No multi-action apply.
- No stop or restart command.
- No image replacement, port mutation, mount mutation, rollback, or broad lifecycle management.
- No image cleanup, volume cleanup, or unmanaged cleanup.
- No Kubernetes, CRI, Docker API, or Docker Compose compatibility.
- No DNS, tunnels, cloud control plane, GPU/ANE scheduling, privileged helper, installer, signing, or notarization.
